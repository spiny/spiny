# Spiny

**A local-first thoughts repository for Android and iOS.**

[![License](https://img.shields.io/badge/license-Apache%202.0-blue.svg)](LICENSE)
[![Platform](https://img.shields.io/badge/platform-Android%20%7C%20iOS-19193b.svg)](#requirements)
[![Expo SDK](https://img.shields.io/badge/Expo%20SDK-56-000020.svg)](https://docs.expo.dev/)

</div>

Spiny is a quick Markdown capture tool, conceptually close to Obsidian but intentionally leaner: catalog-scoped search, local persistence by default, and optional user-owned sync providers. It is built with Expo and React Native and runs entirely offline unless you connect a sync provider.

## Features

- **Local-first storage** — SQLite (`expo-sqlite`) with transactional autosave; your notes work fully offline.
- **Markdown editor** — toolbar (bold, italic, headings, rules, document/external links), a separate preview, and a 64 KB-per-document limit with warnings.
- **Catalogs** — collections of documents, each with its own searchable index and sync configuration.
- **Mind map** — a navigation graph with multi-select copy/move across documents.
- **Search** — catalog-scoped, full-text, offline.
- **Themes and locales** — system/light/dark and full English + French translations.
- **Optional sync** — Google Drive (OAuth PKCE, user-supplied client id, no bundled secrets); SSH/SFTP and FTP are scaffolded but non-functional in v1.

See [`docs/`](docs/README.md) for product and technical documentation.

## Requirements

- **Node.js** 20+ and npm
- **JDK 17** — required by the Expo/Android Gradle toolchain (newer JDKs break the CMake step). Get [Temurin 17](https://adoptium.net/temurin/releases/?version=17).
- **Android SDK** with `adb` on your `PATH` (Android Studio or command-line tools), and `ANDROID_HOME`/`ANDROID_SDK_ROOT` set.
- An Android device with **USB debugging** enabled (or an emulator).
- For iOS: macOS with Xcode (not covered below; Android is the primary target).

> The native `android/` and `ios/` folders are generated by Expo ([Continuous Native Generation](https://docs.expo.dev/workflow/continuous-native-generation/)) and are not committed. The release script runs `expo prebuild` automatically when they are missing.

## Getting started

```bash
git clone git@github.com:spiny/spiny.git
cd spiny
npm install
```

The Spiny scripts locate JDK 17 automatically (checking `JAVA_HOME`, `~/.local/jdks`, `/usr/lib/jvm`, SDKMAN, and Homebrew). If detection fails, set it explicitly:

```bash
export SPINY_JDK17_HOME=/path/to/jdk-17
```

## Run on a device (development, fast refresh)

Connect a device (`adb devices` should list it), then:

```bash
npm run android:dev
```

This compiles the debug build, installs it, and starts the Metro dev server so edits hot-reload on the device.

## Build & install a release APK on a device

This produces a standalone, signed APK that runs **without** the Metro server — the same flow validated on a Moto G Play (Android 11):

```bash
npm run android:release
```

The script resolves JDK 17, runs `expo prebuild` if needed, then builds and installs via Gradle. After changing icons, `app.json`, or plugins, force a fresh prebuild so native resources regenerate:

```bash
SPINY_PREBUILD=1 npm run android:release
```

Target specific ABIs (default is `arm64-v8a`):

```bash
SPINY_ANDROID_ARCH="armeabi-v7a,arm64-v8a,x86,x86_64" npm run android:release
```

The equivalent raw commands (what the script runs) are:

```bash
npx expo prebuild -p android
cd android
JAVA_HOME=/path/to/jdk-17 ./gradlew :app:installRelease -PreactNativeArchitectures=arm64-v8a
```

Launch it afterwards with:

```bash
adb shell monkey -p com.spiny.app -c android.intent.category.LAUNCHER 1
```

> The release variant is signed with the debug keystore for convenient local installs. Generate your own keystore before distributing to the Play Store — see the [React Native signing guide](https://reactnative.dev/docs/signed-apk-android).

## App icons

All launcher, splash, and favicon assets are generated from a single master, `assets/icon-source.png`:

```bash
npm run icons          # regenerate assets/images/* with ImageMagick
npm run prebuild       # apply them to the native projects
```

Requires ImageMagick v7 (`magick`). The design is a dark `#19193b` background with a gold ring/frame; the hedgehog artwork is left unmodified.

## Scripts

| Command | Description |
| --- | --- |
| `npm start` | Start the Expo dev server (choose a target). |
| `npm run android:dev` | Debug build + install + Metro fast refresh on a device. |
| `npm run android:release` | Build and install a standalone release APK on a device. |
| `npm run prebuild` | Regenerate the native `android/` and `ios/` projects. |
| `npm run icons` | Regenerate app icons from `assets/icon-source.png`. |
| `npm run typecheck` | `tsc --noEmit`. |
| `npm run lint` | `expo lint`. |
| `npm run web` | Run the web build locally. |
